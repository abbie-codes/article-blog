5 tips for Responsive Web Design in 2022

BY ABIGAIL UNWIN

April, 11, 2022

5 mins to read

Gone are the days where websites were only accessed on computer screens. It is estimated that half of all internet users in 2022 access the web from their mobile phones, and this number will likely increase, it is predicted that almost three quarters of internet users will be mobile-only by 2025. Google also favours mobile-responsive websites in their search algorithm, so it is essential that web designers ensure that their site looks great and is functional on all screen sizes. This is where responsive web design comes in.

What is responsive design?

Responsive web design allows the code to respond to the size of the device’s screen.

If you look at a static website on a mobile the text is small and ineligible, when you look at a responsive site it has been coded so that the text is larger and easily readable. Images on a responsive web page will be resized to fit onto a smaller screen so that the image can be viewed without scrolling or zooming in. Navigation bars work well on static websites but menus can take up too much space on a mobile sight, these are often replaced with menu icons in the top corner that are clickable with one hand, referred to as hamburger menus because of their circular shape with 3 lines in the middle.

Responsive design was created to give the user optimal viewing experience, whether you are looking at a mobile phone, a tablet or a large monitor.


Here are 5 tips to get you started:



1. Relative length units

Make sure that the settings you use for length are responsive. Absolute units (like pixels) will not allow your website to respond to different device sizes. Instead, use viewport-relative units (like vh or vw) or font-relative units (em or rem). This will allow the elements to scale.

"Absolutely no absolute units!"
- web designer Nick Gard

2. Think about layout first

The layout of your site is important for functionality, user experience and look and feel of the page. Incorporate flexible grids that adjust depending on screen size so that as the viewport narrows the content is more vertical. This will allow your web page to change without sacrificing any content. Setting minimum and maximum values for height and width properties can ensure elements are controlled.

Consider designing at least 3 layouts for different browser widths:

  • Small: Under 600px. This is how the content will look on the majority of mobiles.
  • Medium: 600px – 900px. This is how the content will look on most tablets, some large phones and small laptops.
  • Large: Over 900px. This is how the content will look on most computers.

3. Use Media Queries

So, you have visualised how you want your site to look on mobiles, tablets and computers, how do you make sure your CSS can adapt to different screen sizes? This is where media queries come in.

The @media rule is used to set rules for different screen sizes. Media queries can be used to transform many things such as width and height of device and orientation so you can have different site settings for portrait or landscape mode. You can also use media queries to specify that certain styles are only for printed documents or for screen readers (mediatype: print, screen, or speech).

4. Typography

Make sure that your font is legible across devices. Although there's no official minimum size font for website use, it's usually recommended that you use at least 16px font for the body text, this is a good starting point for the body of your page and then you can adjust headings accordingly. Consider also adjusting the line height of your text so that the paragraph lines are not cramped on the page. Another tip is to use font that is easy to read, especially for the main body of text or menu options a decorative font might be readable on a large screen but hard to read on a mobile device.

5. Test out your webpage

You’ve implemented all of the above steps and think that your webpage will look great and work seamlessly on screens of all sizes, but you aren’t sure how to test it out? Thankfully there are loads of great resources out there on the web that let you test your work!

Here are a few suggestions:

  • Screenfly

    Simply enter the web address, click the display mode you want to view and Screenfly will show you how the webpage will look!
  • MobileTest.me

    This site is specifically for viewing the webpage on different mobile devices. It works similarly to Screenfly, simply select the mobile device you want to use and type in the URL. As simple as that!
  • Google Mobile-Friendly Test

    If you are still not sure if your site is mobile friendly Google have a helpful tool. You can give this webpage a URL or add your code and Google will tell you whether or not the site is mobile friendly. If not Google will let you know where you are going wrong and give suggestions on how to adapt the page.

You are now set to create a beautiful responsive website of your own. We hope you have enjoyed learning these 5 tips and will implement them in your work. Happy Coding!
